# Ordo v0.12 Design Note: Reliability, Trust Semantics & Execution Modes

## 1. Problem

До v0.12 Ordo описувала gates, debug trace і test layer як контрольні механізми, але не розділяла достатньо явно:

- механічні перевірки, які може виконати код;
- семантичні судження, які виконує модель;
- повторні модельні перевірки;
- рішення людини;
- реальний runtime-log;
- самозвіт моделі про виконання.

Це створює ризик завищеної довіри до результату. Наприклад, gate, який рахує кількість пунктів, і gate, який оцінює фактологічну підтримку тверджень, не можуть мати однаковий клас довіри.

## 2. Design decision

Ordo v0.12 вводить reliability-семантику як частину мови:

```text
gate.method + trust_class + trace_source + execution_mode + ASSERTION + control_level
```

## 3. Scope

Ця версія не описує платформні інтеграційні механізми або API-специфічні реалізації. Вона описує тільки внутрішню модель Ordo.

## 4. Core rule

Ordo не повинна створювати ілюзію, що всі перевірки однаково детерміновані. Кожна перевірка має показувати, хто її виконує, яким методом і який клас довіри має результат.
