# M74 — Language Trust, Registry, and Public Baseline Hardening

## Accepted audit findings

1. CSG needs empirical model benchmarking distinct from specification consistency.
2. New language constructs need canonical registry and deterministic completeness gates.
3. Public language baseline, process archive, and book need separate release artifacts.
4. APF must become the principal modularity dogfooding test after include/registry hardening.
5. Development-workspace hygiene and release-archive hygiene must be tested separately.
