# Test and improve prompt

Test the draft playbook before finalizing it.

Use at least these cases:

1. all required weekly notes are present;
2. blockers are explicitly “none”;
3. next steps are missing;
4. the notes contain an unsupported claim;
5. the user rejects the first final report.

For each case, show the expected route, gate result, and output behavior. Identify any ambiguity, skipped gate, invented fact, or irreversible step. Propose the smallest correction, explain its effect, and ask for approval before changing the playbook.

After approval, rerun the affected cases and return the final package inventory.
