# APF Conversation Scope Guard — State Protection and Process Control Policy

## Boundary

This policy applies only to playbooks generated by APF when Conversation Scope Guard is explicitly enabled. APF does not enable CSG for its own authoring conversation.

## Core rule

Classification occurs before any state mutation. `unrelated_topic` and `unclassifiable_input` cannot complete the active node, select a path, confirm a value, or clear collected state.

## Process-control intents

Pause, resume, exit, correction, backtracking, requirement change, and safety messages are not ordinary deviations. They bypass unrelated-topic handling and are routed to their existing process-control contracts.

## Pause

Pause preserves the active state and node, records the reason, and changes only execution status to `paused`. It does not imply success or approval.

## Resume

Resume restores the preserved state and returns to the suspended node or an explicitly declared resume node. It cannot skip unresolved gates or approvals.

## Exit

Exit is a controlled terminal transition. When the process contract is incomplete, the terminal status is `exited_incomplete`, never `completed`.

## Validation boundary

Generated-playbook release is blocked when any state-protection assertion is absent, control operations have undefined preconditions, or safety/control intents can be rejected as unrelated.
