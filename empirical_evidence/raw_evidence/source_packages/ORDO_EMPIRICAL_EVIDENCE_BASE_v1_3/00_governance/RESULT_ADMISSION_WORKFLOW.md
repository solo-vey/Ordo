# Result Admission Workflow

For every new run:

1. Identify implementation, run ID, scenario, and package version.
2. Verify archive integrity and inspect primary evidence.
3. Distinguish model behavior from Driver or evaluator defects.
4. Present factual findings.
5. Present six component scores and the weighted total.
6. Propose a verdict and evidence polarity.
7. Ask for explicit admission confirmation.
8. On confirmation, store the raw archive, English metadata, scorecard, and register entry.
9. Update the comparative case report and checksums.
