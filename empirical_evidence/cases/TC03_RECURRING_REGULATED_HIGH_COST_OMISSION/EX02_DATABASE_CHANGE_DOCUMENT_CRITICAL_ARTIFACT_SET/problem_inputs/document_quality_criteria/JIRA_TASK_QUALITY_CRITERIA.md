# Jira Task Quality Evaluation Criteria

The Jira task must be a complete handoff for manager, developer and tester.

Required: summary; context/current state; detailed atomic acceptance criteria; data mapping; technical deliverables; concrete unit/processor test requirements; QA reference data; manual QA case summary with IDs, data and expected results; automation reference and expected coverage; implementation constraints; out of scope; companion-artifact references.

Acceptance criteria must be testable. “Add tests”, “support the change” and “update documentation” are insufficient. Technical deliverables must identify components, tests, docs, registries or artifact categories to check/update.

Failure cap: a Jira task without concrete test cases, technical deliverables, QA reference data and detailed acceptance criteria cannot score above 50%.
