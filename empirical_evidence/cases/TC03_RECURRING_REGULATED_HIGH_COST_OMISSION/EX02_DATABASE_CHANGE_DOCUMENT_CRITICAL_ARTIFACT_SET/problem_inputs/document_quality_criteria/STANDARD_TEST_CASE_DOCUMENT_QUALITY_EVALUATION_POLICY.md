# Standard Test-Case Document Quality Evaluation Policy

For every future test case, define input-side quality criteria for the passport, Jira task, manual QA package and QA automation specification before execution.

Final run score:
- 50% process/instruction execution quality;
- 50% generated-document quality.

Correct terminal behavior does not make a run successful when principal output documents are incomplete, shallow or non-executable. File existence and superficial structure are insufficient.
