# Rollback and Release Discipline

Before material version changes, the prior package was frozen, checksum-addressed, archived, extracted into a disposable directory, and restore-verified. The release candidate was not promoted until structural checks, semantic checks, tests, checksums, archive integrity, rollback restoration, and clean post-unpack validation were green.

The actual rollback archives are not redistributed here because this publication package is intended to minimize unnecessary duplication and source exposure. Their existence and verified outcomes are recorded in the release reports and development evidence.
