# Architecture

Flow: intake → relevance → data gates → identity → three user planes → bilingual trigger/placeholder contract → scenarios → artifacts → acceptance → sign-off → external links → final validation. Terminal outcomes are business-ready, blocked, not-relevant, undeliverable, and cancelled.


## Two-stage package lifecycle

The first artifact set is a reviewable draft. External systems are changed manually by the analyst only after content approval. Returned external documentation system and work-tracking system URLs are then synchronized into a new package revision, followed by stale-link and final readiness validation.
