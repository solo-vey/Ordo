# Confidentiality and IP Policy

## Exclusion policy

This package must not contain:

- original uploaded instruction documents;
- original framework source archive;
- original event-specific reference documents;
- original answer scripts;
- original run ZIP files;
- exact business aliases, templates, payload paths, examples, or taxonomy copied from confidential inputs;
- real URLs, issue keys, organization names, personal names, or contact details.

## Allowed evidence

- independently created playbook implementation;
- generalized descriptions of input categories;
- non-reconstructive chronology;
- abstracted defects and fixes;
- validation results;
- synthetic examples;
- package hashes and release metadata.

## Publication caveat

Absence of original inputs means this package proves the development process and produced implementation, not full fidelity to the confidential business specification.
