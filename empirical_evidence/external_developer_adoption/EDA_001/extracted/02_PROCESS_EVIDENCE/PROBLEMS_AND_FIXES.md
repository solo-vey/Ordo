# Problems Encountered and Resolutions

## P-001 — Source document initially treated too much like package source
**Problem:** the confidential all-in-one process description was initially interpreted as a possible maintained instruction source.
**Fix:** reclassified it as requirements evidence only and prohibited its inclusion in deliverables.

## P-002 — External lifecycle modeled as fields rather than process
**Problem:** early design captured external URLs but did not fully represent ordered human publication steps.
**Fix:** added draft review, first external publication, URL capture, second document generation/publication, second URL capture, regeneration, and final validation nodes.

## P-003 — Identity inconsistency in pilot output
**Problem:** one generated draft used different values for the same group identity in two artifacts.
**Fix:** returned the draft for targeted correction and strengthened identity consistency validation.

## P-004 — Unsupported open question invented by executing model
**Problem:** the pilot introduced a retention question not present in the supplied answers or contract.
**Fix:** removed the question and treated unsupported uncertainty creation as a non-fabrication defect.

## P-005 — Semantic rule passed but machine assertion failed
**Problem:** a human-authority policy was semantically correct but did not match the exact validator assertion.
**Fix:** aligned the normative wording and reran checks. This showed that machine-testable contract language must be explicit.

## P-006 — Sign-off evidence too aggregate
**Problem:** a final package contained a completion flag without enough per-person evidence.
**Fix:** execution evidence later demonstrated separate observable confirmations, while the generic structured-signoff requirement was moved to the framework backlog.

## P-007 — Backlog over-engineering
**Problem:** several proposed domain enhancements were not required by the actual business process.
**Fix:** removed them, retained only lifecycle/link-placement changes, and deferred framework migration.
