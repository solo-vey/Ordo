# Для кого ця книга

![Nebu — це цікаво](../assets/mascots/64x64/Nebu_interesting_64x64.png)

Ця книга для людей, які вже використовують AI-моделі не тільки для простих відповідей, а для складних робочих процесів.

Вона буде корисною для аналітиків, які описують playbook-и, правила перевірки, сценарії збору даних і пакети документів. Вона також буде корисною для розробників, які хочуть перетворити AI-інструкції на щось більш стабільне, тестоване і придатне для підтримки.

Окрема аудиторія цієї книги — автори внутрішніх стандартів, QA-фахівці, product managers, технічні письменники і всі, хто стикався з ситуацією, коли модель “майже все зрозуміла”, але пропустила один критичний крок.

Для читання не потрібно бути програмістом. Але буде корисно мати базове розуміння того, що таке інструкція, перевірка, стан процесу, результат і помилка виконання.

Ця книга також для тих, хто хоче в майбутньому публікувати Ordo-програми, бібліотеки, domain packs або playbook-и як відкриту документацію на GitHub.
