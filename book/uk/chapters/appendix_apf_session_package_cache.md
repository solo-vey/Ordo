# Додаток: одноразове завантаження пакета в APF

APF не повинен повторно розпаковувати й перечитувати незмінний пакет на кожному кроці. Після першого успішного завантаження зберігаються version, SHA-256 fingerprint, шлях до cache та статуси manifest/source-of-truth.

Перед виконанням вузла `PACKAGE_RELOAD_NECESSITY_GATE` або повертає `PACKAGE_CACHE_HIT`, або дозволяє reload лише при реальній зміні пакета, втраті cache, відсутності файла, помилці валідації чи явній команді користувача. Перевірка cache не змінює active node і не просуває процес.
