# Вступ. Головна ідея Ordo

![Nebu — ідея](../assets/mascots/64x64/Nebu_idea_64x64.png)

Головна ідея Ordo полягає в тому, що складна інструкція для AI-моделі має бути не просто текстом, а виконуваним контрактом.

Звичайний prompt часто описує бажаний результат. Ordo описує шлях до результату: що потрібно зʼясувати, які рішення можна приймати, де треба зупинитися, які перевірки пройти, що вважати готовим результатом і як пояснити виконання.

Базова формула Ordo виглядає так:

```text
intent → contract → context → state → path → steps → gates → result → handoff
```

Ця формула важлива, бо вона змінює сам підхід до роботи з моделлю. Ми більше не просто просимо модель “зробити добре”. Ми визначаємо процес, у межах якого модель має працювати.

Ordo не намагається зробити AI-модель звичайною детермінованою програмою. Модель усе одно залишається семантичним виконавцем: вона розуміє текст, узагальнює, формулює, порівнює, пропонує. Але Ordo задає межі цієї свободи.

У короткій формі:

```text
Ordo — це мова керованого виконання інструкцій для AI-моделей.
```

Або ще простіше:

```text
Ordo перетворює людський намір на процес, який модель може виконувати, перевіряти, пояснювати і покращувати.
```

## M30 уточнення: Ordo як Process Rail

Поточна головна ідея Ordo: це не CLI-first runtime і не жорсткий wizard. Ordo дає ШІ Process Rail — опорну структуру, яка тримає стан, маршрут, gates, повернення назад і результат, але залишає ШІ активним мислячим виконавцем.

```text
ШІ мислить і спілкується.
Process Rail тримає процес.
Semantic JSON IR зберігає машинозчитувану форму рейки.
CLI перевіряє детерміновані частини.
```
