# M46.7 Source archive hygiene

M46.7 makes the pre-release source archive stricter without changing the Ordo language model.

The source workspace should contain authoring inputs, specifications, package sources, tests and docs. It should not contain generated helper outputs from local runs.

`ordo repo-check` now verifies three repository-level checks:

1. GitHub workflow package paths exist.
2. Python generated metadata is absent (`__pycache__`, `*.pyc`, `*.egg-info`).
3. Package-generated helper artifacts are absent from source archives, except `.gitkeep` placeholders.

Forbidden generated package artifacts include files under:

```text
packages/*/compiled/
packages/*/reports/
packages/*/runtime/
packages/*/generated_outputs/
```

These files are intentionally regenerated by CLI commands such as:

```text
ordo compile
ordo test
ordo coverage
ordo intake
ordo generate-output
ordo validate-artifacts
ordo consistency
ordo go-no-go
```

This keeps the repository honest: validation reports and runtime traces should be evidence produced by the current run, not stale files carried from an older run.
