# Start here

This Kit is for creating one new Ordo playbook in a language-model chat.

1. Upload the complete ZIP to a new chat.
2. Ask the model to read every file in the archive before responding.
3. Paste `START_PROMPT.md` into that chat.
4. Describe the process you want to create or improve.
5. Review the draft, dry-run, test results, and proposed improvements before
   approving a final package.

You do not need Python, a terminal, or a repository clone for this route.

Read the files in this order:

1. `PLAYBOOK_LAWS.md`
2. `START_PROMPT.md`
3. `PLAYBOOK_TEMPLATE.md`
4. `PLAYBOOK_BRIEF.md`
5. `TEST_AND_IMPROVE.md`
6. `EXPECTED_DELIVERABLES.md`

Conversational review is useful for drafting and learning. If deterministic
validation is unavailable, the final result must say
`conversational_validation_only` and must not claim release-grade evidence.
