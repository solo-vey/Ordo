# L documentation contour closure

Status: `closed`

The L contour establishes chat-based use as Ordo's primary onboarding path. Python and the CLI remain optional deterministic helpers for automation, reproducibility, CI, and release-grade validation.

The contour completed:

- the L.1 read-only documentation quality-gate audit;
- the L.2 chat-first starter, quickstart, and current-state synchronization;
- the L.3 repository, documentation, release-candidate, and checksum validation;
- the L.4 final evidence, backlog, maturity-state, and contour closure.

The reproducible release builder is the canonical final-archive mechanism. Generated release archives remain outside Git and are accepted only after delivery and strict release gates pass.

Historical and localized documents outside the active contour retain their original bytes. The preserved migration debt is explicitly listed in the [machine-readable closure record](../../manifests/L_DOCUMENTATION_CONTOUR_CLOSURE.json) and does not define the current documentation front door.

Start with the [chat-first onboarding guide](../CHAT_FIRST_ONBOARDING.md), the [quickstart](../QUICKSTART.md), or the [starter package](../../examples/chat_first_playbook_starter/README.md).
