# DD-ORDO-M67-004 — Clean Package Gate Alignment to Implemented CLI

Status: accepted
Date: 2026-07-09

## Context

M67.2 defined the `ordo clean-check` command contract. M67.3 implemented a minimal deterministic CLI command. The language package now needs to align clean package gate documentation and schemas to the implemented behavior.

## Decision

Add language-level documentation and schema conventions for:

- `clean_package_gate`;
- `derived_artifact_sync_validation_profile`;
- clean-check report interpretation;
- derived artifact / delta backlog severity mapping.

The canonical executable evidence remains the CLI report generated by `ordo clean-check`.

## Non-decision

This decision does not introduce:

- package-local patches;
- runtime behavior;
- compiler behavior;
- opcodes;
- automatic artifact regeneration;
- lockfile regeneration;
- embedded CLI rebuilds;
- deterministic natural-language authority classification.

## Consequences

Language documentation can now describe clean package readiness as an implemented CLI-backed gate rather than as a future-only design concept.
