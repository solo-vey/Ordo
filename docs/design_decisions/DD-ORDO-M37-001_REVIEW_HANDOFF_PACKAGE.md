# DD-ORDO-M37-001 — Review Handoff Package

## Status

Accepted.

## Decision

M37 adds a reviewer handoff layer on top of the frozen `v0.12.0-preview` release candidate. It does not introduce new language/runtime functionality.

## Reason

After the M36 freeze, the next useful step is not new feature work but a clear review route for an external reviewer. Without a handoff package, the reviewer would have to reconstruct the story from many milestone artifacts.

## Scope

In scope:

- reviewer route;
- reviewer questions;
- handoff summary;
- publication manifest update;
- validation report.

Out of scope:

- changing the license mode;
- implementing new runtime features;
- changing package structures;
- claiming open-source release status.

## Impact

This is a documentation/release-evidence milestone. It preserves the M36 freeze and improves reviewability.
